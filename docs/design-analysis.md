# Анализ двух дизайнов: AURA Book vs Poor Charlie's Almanack

## TL;DR

**Дизайн 1 (AURA Book)** — клиент считает его **плохим** (точнее, «технически правильным, но эмоционально скучным»).  
**Дизайн 2 (Poor Charlie's Almanack, Stripe Press)** — клиент считает его **идеалом**, к которому нужно стремиться.

Ирония в том, что клиент **не хочет копировать ни один из них**. Он хочет взять **хореографию и живость** от PCA + **чистоту и дисциплину** от Linear/Notion + **ощущение книги** (а не SaaS). Ниже — по-человечески, что это значит.

---

## 1. Дизайн AURA Book — что сделано и почему клиент недоволен

### Что там есть
По описанию в брифе и из Figma-прототипа, AURA Book — это:

- **Трёхколоночный layout**: узкая левая навигация (лого + ссылки), средняя колонка (оглавление текущей Части), правая колонка (текст главы).
- **Нейтральная серая палитра** — много серого, мало контраста, мало жизни.
- **«Плавающий белый прямоугольник» (island)** — текстовый блок визуально «подвешен» в центре экрана на сером фоне, как карточка в дашборде.
- **Шрифты** — клиенту не нравятся ни заголовочный, ни основной.
- **Оверлей оглавления** — функционально правильный (поиск, список глав).

### Почему это «плохо» — 5 конкретных проблем

| Проблема | Суть |
|----------|------|
| **🎨 Серость** | Палитра выглядит как админка SaaS-приложения. Нет тепла, нет «приглашения почитать». Открываешь — и ощущение: «ещё один дашборд». |
| **📦 Island-подход** | Белая карточка на сером фоне — это паттерн из Material Design, Google Docs, Notion-editor. Он говорит мозгу: «ты сейчас будешь работать», а не «ты сейчас будешь читать книгу». |
| **🔤 Типографика без характера** | Шрифты не несут ощущения «серьёзного автора с позицией». Они нейтральные до безликости — как системный шрифт. |
| **📐 Статичность layout-а** | Текст идёт одним непрерывным потоком в одной колонке. Нет ритма, нет момента, когда страница «оживает» при появлении иллюстрации. |
| **🖼 Картинки как вставки** | Изображения просто встроены в поток текста, как в Medium-посте. Они не «разговаривают» с текстом — они просто стоят между абзацами. |

### Одной фразой
> AURA Book выглядит так, будто его делали по чеклисту UI-компонентов, но забыли про эмоцию. Он **правильный**, но **мёртвый**.

## 2. Poor Charlie's Almanack (Stripe Press) — что делает его «идеалом»

### Технический стек и палитра
Из CSS-кода PCA видно:

- **Фон**: `#EFEFEF` — это не чистый белый, а тёплый светло-серый. Тонкость: это не «серая админка», а «бумага». Мозг считывает это как книжную страницу.
- **Шрифты**:
  - **Cardinal Fruit** — заголовочный/акцентный serif с характером (изящный, не банальный).
  - **Ivar** — body-text serif, созданный для долгого комфортного чтения.
  - **Suisse Mono** — для технических элементов.
  - Все три — **премиальные платные шрифты**. Это важно: PCA может себе позволить $500+ за лицензии, а твоему клиенту нужны бесплатные аналоги.
- **Выделение текста**: при selection текста фон становится `#2d2d2d` (тёмно-серый) с белым текстом — маленький, но важный штрих, который говорит «здесь думали о каждой детали».

### Пять вещей, которые делают PCA волшебным

#### 2.1. «Хореография» между текстом и картинками

Это ключевое отличие. PCA **не просто вставляет картинки в поток текста** — страница **трансформируется**:

```
Обычный режим:          Двухпейновый режим:
┌─────────────────┐     ┌────────┬──────────┐
│                 │     │        │          │
│    Текст        │     │ Картин-│  Текст   │
│    Текст        │  →  │ ка     │  Текст   │
│    Текст        │     │        │  Текст   │
│                 │     │        │          │
└─────────────────┘     └────────┴──────────┘
```

Страница **расщепляется** на два pane-а, когда встречается визуально сложный концепт, и **собирается обратно**, когда иллюстрация не нужна. Это создаёт **ритм чтения** — как в настоящей красивой книге, где разворот с иллюстрацией чередуется с текстовыми разворотами.

#### 2.2. Pull-quotes — типографические акценты

PCA использует **крупные курсивные serif-цитаты**, которые разрывают поток текста:

> *"The safest way to get what you want is to try and deserve what you want."*

Это не просто `<blockquote>` с левой полоской. Это **архитектурный элемент** — как большая буквица (drop cap) в печатных книгах. Он говорит: «Остановись, подумай». Он создаёт **визуальную паузу** в ритме чтения.

#### 2.3. Фон как бумага, не как экран

PCA использует `#EFEFEF` — не белый, а **тёплый off-white**. Это кажется мелочью, но мозг считывает:

- `#FFFFFF` (чистый белый) → «экран монитора, приложение»
- `#EFEFEF` / крем / слоновая кость → «бумага, книга, физический объект»

При этом PCA **не уходит в сепию** (как винтажные сайты) — тон нейтральный, современный.

#### 2.4. Типографическая иерархия без визуального шума

PCA не использует цветные плашки, иконки, бейджи. Вся иерархия — через:
- **Размер шрифта** (маленькие подписи vs огромные pull-quotes)
- **Вес** (Regular vs Medium)
- **Стиль** (Roman vs Italic)
- **Whitespace** (отступы между блоками)

Это создаёт ощущение **дисциплины и уверенности** — как будто дизайнер знает, что ему не нужны костыли в виде цветных рамок, чтобы направить внимание читателя.

#### 2.5. Щедрый whitespace

В PCA **много воздуха**. Между абзацами, вокруг картинок, между pull-quote и следующим абзацем. Это не «пустое место» — это **тишина между нотами в музыке**. Без неё текст был бы стеной.

## 3. Главные различия — таблица

| Аспект | AURA Book (❌ текущий) | PCA / Stripe Press (✅ идеал) |
|--------|----------------------|-------------------------------|
| **Фон** | Чистый серый → «дашборд» | Тёплый off-white → «бумага» |
| **Layout** | Статичный, одна колонка текста | Динамичный: страница «расщепляется» и «собирается» |
| **Картинки** | Вставлены в поток, как в блог-посте | Танцуют с текстом, создают хореографию |
| **Типографика** | Безликая, без характера | Serif с личностью, иерархия через размер/вес |
| **Pull-quotes** | Нет или стандартный blockquote | Крупный курсивный serif, архитектурный элемент |
| **Whitespace** | Нормальный (стандартные отступы) | Щедрый, «текст дышит» |
| **Ощущение** | «Открыл документ в Google Docs» | «Открыл красивую книгу на хорошей бумаге» |
| **Эмоция** | Нейтральность → скука | Спокойная уверенность → уважение к читателю |
| **Island** | Белая карточка на сером фоне | Нет острова — весь фон однородный |

## 4. Что хочет клиент — простыми словами

Клиент хочет **невозможного (но красивого) гибрида**. Вот формула:

```
PCA-хореография  +  Linear/Notion-чистота  +  ощущение книги  =  цель
(layout, ритм)     (минимализм, дисциплина)   (не SaaS, не дашборд)
```

### Берём от PCA:
- ✅ Как картинки и текст живут вместе (двухпейн, хореография)
- ✅ Pull-quotes как архитектурные элементы
- ✅ Ритм страницы: single-pane → dual-pane → single-pane
- ✅ Типографическая иерархия без визуального мусора

### НЕ берём от PCA:
- ❌ Винтажный стиль (сепия, старомодный serif, коллажи)
- ❌ Конкретные платные шрифты (Cardinal Fruit, Ivar)
- ❌ Историческую тематику

### Берём от Linear/Notion:
- ✅ Чистота и дисциплина сетки
- ✅ Современный sans-serif (Inter, Geist, IBM Plex Sans)
- ✅ Миллиметровая точность отступов
- ✅ Сдержанная, неяркая палитра

### НЕ берём от Linear/Notion:
- ❌ Тёмную тему
- ❌ Ощущение «инструмента для работы»
- ❌ Dashboard-паттерны (карточки, бейджи, прогресс-бары)

### Добавляем сверху:
- ✅ Тёплый фон (не белый, не серый → «бумага»)
- ✅ Serif для body-текста (или хотя бы для pull-quotes) — для «книжности»
- ✅ Щедрый whitespace — «текст дышит»
- ✅ Голос Скептика — визуально узнаваемый персонаж в тексте

## 5. Метафора, которая всё объясняет

> **AURA Book** — это IKEA-полка для книг: функциональная, чистая, но ты не замечаешь её. Она фон.
>
> **PCA** — это книжный магазин с деревянными стеллажами, где пахнет кофе и бумагой. Ты остаёшься дольше, чем планировал.
>
> **Клиент хочет** — Apple Store для книг: минималистичный, современный, чистый до блеска, но при этом ты чувствуешь, что каждый предмет здесь положен с уважением и любовью. Спокойная мощь.

## 6. Конкретные рекомендации для конкурса

### 6.1. Палитра
- Фон: **не белый, не серый** → тёплый off-white / ivory / мягкий крем (`#F5F3EF` — `#FAF9F6` диапазон)
- Текст: `#1A1A1A` — почти чёрный, но не `#000`
- Акцент: **один цвет**, сдержанный — тёмно-синий / оливковый / терракотовый. Не яркий.
- Серый для мета-информации: `#6B7280` — `#9CA3AF`

### 6.2. Типографика
- **Заголовки**: современный sans-serif с характером — **Outfit**, **Space Grotesk**, **Sora** (Google Fonts, бесплатные)
- **Body**: читабельный serif для книжности — **Source Serif 4**, **Crimson Pro**, **Literata** (все бесплатные)
- **Pull-quotes**: крупный italic serif — тот же body-шрифт в italic, но 24-28px
- **Code/meta**: **JetBrains Mono** или **IBM Plex Mono**

### 6.3. Layout — ключевой приём
Реализовать **динамический layout**, который переключается:

```
┌──┬────────┬──────────────────┐   ← обычный режим: навигация + оглавление + текст
│  │ TOC    │                  │
│  │        │  Текст главы     │
│  │        │                  │
│  │        │  ───────────     │
│  │        │                  │
│  │        │  [картинка]      │   ← картинка в потоке — full-width текстовой колонки
│  │        │                  │
│  │        │  Текст...        │
└──┴────────┴──────────────────┘

┌──┬────────┬────────┬─────────┐   ← двухпейн: когда нужна иллюстрация к тексту
│  │ (скры- │        │         │
│  │ вается │ Картин-│  Текст  │   ← средняя колонка (TOC) может сужаться/скрываться
│  │ или    │ ка     │  рядом  │      чтобы дать место двухпейну
│  │ сужа-  │        │  с ней  │
│  │ ется)  │        │         │
└──┴────────┴────────┴─────────┘
```

Если получится сделать хотя бы **50% этого** — клиент будет в восторге. Цитата из брифа: *«Если получится приблизиться к этой модели хотя бы наполовину — это уже большая победа.»*

### 6.4. Скептик — узнаваемый голос
Это уникальный элемент книги. Предложения:
- Тонкая цветная полоса слева (не серая — акцентная, но приглушённая)
- Иконка «?» или стилизованный аватар
- Шрифт italic + фон чуть отличающийся от основного
- Обязательно должен «стоять» иначе, чем обычный blockquote

## 7. Резюме для тебя

Когда будешь делать дизайн, держи в голове три вопроса:

1. **«Это похоже на приложение или на книгу?»** — должно быть на книгу.
2. **«Картинка здесь стоит или живёт с текстом?»** — должна жить.
3. **«Если убрать весь контент, какое ощущение от пустой страницы?»** — должно быть: тёплое, чистое, уважительное. Не серое и не холодное.
