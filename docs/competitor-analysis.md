# Анализ дизайна конкурента

**Сайт:** `editorial-book-ui-41282466630.us-west1.run.app`  
**Стек:** React 19 + Vite + Tailwind CSS v4.3 + Google Fonts  
**Вывод:** Дизайн **сильно лучше** AURA Book, но **всё ещё не дотягивает** до того, что хочет клиент. Ниже — разбор.

---

## 1. Что конкурент сделал технически

### Палитра (из CSS-переменных)
```css
--color-paper:   #f2efe9     /* тёплый off-white — бумажный фон ✅ */
--color-ink:     #18181a     /* почти-чёрный текст ✅ */
--color-ink-muted: #555      /* приглушённый текст для мета ✅ */
--color-border:  #d8d4cc     /* тёплая граница ✅ */
--color-pink:    #eac6c0     /* акцент — розовый/сalmон для selection ✅ */
```

### Шрифты (из @import Google Fonts)
| Роль | Шрифт | Характер |
|------|-------|----------|
| **Serif (body)** | **EB Garamond** (400–700, italic) | Классический книжный serif |
| **Sans-serif** | **Inter** (300–600) | Современный UI-шрифт |
| **Mono** | **Space Mono** (400, 700, italic) | Технический/code |
| **Dyslexic** | **Lexend** (300–600) | Доступность ♿ |

### Кастомные CSS-токены для чтения
```css
--reader-font-size-sm   /* мобильный размер */
--reader-font-size-md   /* десктоп */
--reader-font-size-lg   /* крупный */
--reader-line-spacing    /* межстрочный интервал */
--font-reader           /* переключаемый шрифт */
```

### Ключевые CSS-приёмы
- **Drop cap** (буквица): `.drop-cap:first-letter` — 5rem serif, плавающая слева. Книжный приём ✅
- **Selection color**: `::selection { background-color: var(--color-pink) }` — мягкий розовый ✅
- **Text justify**: класс `.text-justify` для body-текста
- **Hero blend mode**: `.hero-mix { mix-blend-mode: multiply }` — визуальный эффект на обложке
- **Grayscale/sepia фильтры** на изображениях — книжная обработка
- **Rotations** (`.rotate-1`, `-rotate-2`, `rotate-[1.5deg]`) — слегка повёрнутые элементы, «живой» стиль
- **Blur/backdrop-blur** — для модальных оверлеев и навигации

## 2. Что конкурент сделал хорошо ✅

### 2.1. Палитра — попадание в бриф
`#f2efe9` — идеальный «бумажный» фон. Не белый, не серый, не сепия. Ровно то, что клиент просил: *«ощущение бумаги, а не экрана»*. Это **значительно лучше**, чем серая админка AURA Book.

### 2.2. Типографика с настройками
Конкурент реализовал **переключатель шрифтов** (serif ↔ sans ↔ dyslexic) и **настройки размера/интерлиньяжа**. Это показывает серьёзность подхода к чтению. Клиент этого не просил напрямую, но это **хороший бонус**.

### 2.3. Буквица (Drop Cap)
Классический книжный приём. 5rem первая буква serif-шрифтом. Это сразу говорит: *«ты читаешь книгу, а не блог»*.

### 2.4. Service Worker
Кэширование через SW — оффлайн-чтение. Опять не в брифе, но показывает понимание use-case.

### 2.5. Двухколоночный layout
Из CSS видно паттерн split-pane:
- `lg:w-[45%]` / `lg:w-[55%]` — два блока 45/55
- `lg:w-[50%]` — равные половины
- `lg:flex-row` с `lg:border-r` — разделительная линия
- `lg:sticky` + `lg:h-screen` — sticky-панель на всю высоту

Это **попытка реализовать PCA-двухпейн** — изображение слева, текст справа. Очень важный элемент.

### 2.6. Визуальные эффекты на изображениях
- Сепия, grayscale, multiply blend — обработка картинок
- Повороты (`rotate-1`, `-rotate-2`) — «коллажный» стиль
- Тени (`shadow-lg`, `shadow-2xl`) — глубина

## 3. Что конкурент сделал плохо / спорно ⚠️

### 3.1. EB Garamond — плохой выбор для body
EB Garamond — красивый шрифт, но **плохо работает на экране** при длинном чтении:
- Тонкие штрихи теряются при мелких размерах
- Низкий x-height → хуже читается в массе текста
- **Клиент хотел** *«типографику, на которой не устают глаза за час чтения»* — EB Garamond не про это

**Лучшие альтернативы:** Source Serif 4, Literata (специально для экрана), Crimson Pro.

### 3.2. Стиль скорее «editorial-журнал», чем «Linear + книга»

Из CSS видно много **декоративных** приёмов:
- Повёрнутые изображения (`rotate-1`, `-rotate-6`)
- Множество blend-mode эффектов
- Сепия-фильтры
- Коллажный стиль

Это ближе к **PCA-винтажу** или **editorial-журналу** — ровно то, что клиент просил **НЕ делать**:

> *«Чего НЕ заимствуем у PCA: конкретный визуальный стиль (винтажный коллаж, старомодный serif, цвета-сепия)»*
>
> *«Минимализм + дисциплина, не «editorial-журнал» и не «brutalist-эксперимент»»*

Конкурент **скопировал визуальный стиль PCA** вместо его **layout-логики**. Это ключевая ошибка.

### 3.3. Розовый акцент — спорный выбор
`--color-pink: #eac6c0` — мягкий розово-лососёвый. Для selection — ок, но если он используется шире:
- Клиент описывает **«спокойную мощь зрелого профессионала»** — розовый не про это
- Шкала клиента: serious 78%, bold 86%, mature 82% — розовый сдвигает к «мягкому/женственному»
- Лучше бы подошёл тёмно-синий, оливковый, терракотовый

### 3.4. Tailwind CSS — ломает «рукотворный» характер
Весь проект на Tailwind, что видно по utility-классам. Это не проблема сама по себе, но:
- Tailwind-проекты часто выглядят «шаблонно» — utility-first подход не способствует уникальности
- Конкурент **не написал кастомный CSS** для типографики — он использовал tailwind-утилиты для всего
- Клиент хочет *«миллиметровую разметку»* — Tailwind spacing с `var(--spacing) * N` слишком грубый

### 3.5. Нет трёхколоночного layout (?)
Из CSS **не видно** паттерна трёх колонок, который клиент описывает в §7.1:
- Узкая левая (80px) — навигация
- Средняя (~280-320px) — оглавление текущей части
- Правая (~700-800px) — текст

Конкурент, похоже, сделал **двухпанельный** layout (split-view), а не трёхколоночный с боковой навигацией.

### 3.6. title остался «My Google AI Studio App»
```html
<title>My Google AI Studio App</title>
```
Это показывает, что проект **создан через Google AI Studio** и конкурент не потрудился сменить даже тайтл. Это **огромный red flag** для клиента — говорит о невнимательности к деталям.

## 5. Общая оценка конкурента

### Сильные стороны
1. **Палитра идеальная** — попали в бриф по цвету
2. **Двухпейн есть** — попытка реализовать PCA-хореографию
3. **Буквица, настройки чтения, SW** — бонусные фичи
4. **Тёплый, живой** — не мёртвая серая админка
5. **Responsive** — мобилка продумана

### Слабые стороны
1. **Визуальный стиль — editorial/коллаж** вместо Linear/Notion-чистоты
2. **EB Garamond** — красиво, но не для экранного чтения
3. **Розовый акцент** — не «спокойная мощь», а «мягкость»
4. **Нет трёх колонок** — нарушена архитектура §7.1
5. **«My Google AI Studio App»** — халтура в деталях
6. **Копирует PCA визуально**, а не его layout-логику
7. **Скорее всего нет голоса Скептика** — обязательный элемент

## 6. Как тебе его обойти — конкретно

### Что ты можешь сделать лучше:

| Аспект | Конкурент | Твоя возможность |
|--------|-----------|------------------|
| **Стиль** | Editorial/коллаж | **Linear-чистота + книжность** — ровно то, что клиент просил |
| **Шрифт body** | EB Garamond | **Source Serif 4** или **Literata** — читаются лучше на экране |
| **Шрифт heading** | EB Garamond же | **Outfit / Space Grotesk** (sans) — контраст с body serif |
| **Акцент** | Розовый | **Тёмно-синий / оливковый** — «спокойная мощь» |
| **Layout** | Двухпейн | **Трёхколоночный** (§7.1) + динамический двухпейн |
| **Скептик** | Нет/? | **Визуально узнаваемый блок** — конкурентное преимущество |
| **Детали** | «My Google AI Studio App» | **Каждая деталь проработана** — SEO, title, шрифты подписаны |
| **Минимализм** | Декор (повороты, тени, blend) | **Типографическая иерархия** без визуального мусора |
| **Настройки чтения** | ✅ | Если сделаешь тоже — паритет; если нет — не критично |

### Формула победы:

```
Палитру бери у конкурента     → #f2efe9 — они попали
Стиль бери у Linear/Notion    → чистота, дисциплина сетки
Layout бери у PCA             → двухпейн-хореографию
НЕ бери у конкурента          → декоративность, коллажность, розовый
Добавь своё                   → Скептик, трёхколоночность, title/SEO
```

## 7. Метафора-сравнение

> **AURA Book** = офисный IKEA-стол. Функционально, но душа не лежит.
>
> **Конкурент** = винтажная кофейня с крафтовой бумагой на стенах. Тепло и уютно, но немного *too much* — ты пришёл читать серьёзную non-fiction книгу, а вокруг эстетика Instagram-кафе.
>
> **PCA (Stripe Press)** = частная библиотека в доме архитектора. Каждая полка — на своём месте, каждая книга — открыта с уважением.
>
> **Что нужно клиенту** = Apple Store, в котором единственный товар — книга. Минимализм, свет, воздух, но ты чувствуешь: книга здесь — главная.
